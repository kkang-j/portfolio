# 포트폴리오 가이드

## 1. 콘텐츠 원칙

### 숫자 + 역할 + 임팩트 공식
- 구체적인 성과 수치가 반드시 보여야 함
- 예시: "X원 예산으로 MZ 타겟 캠페인 1건 성공시킨 신입 마케터"
- 예시: "전환율 n% 달성, 데이터로 말하는 그로스마케터"
- 숫자 없는 설명은 임팩트 없음

### 타겟 + 액션 + 특기 공식
- 회사가 타겟하는 고객층과 내 경험이 매칭되는지 확인
- 지원 직무·회사에 따라 강조 포인트 조정

---

## 2. 구성

| 슬라이드 | 내용 |
|---|---|
| 표지 | 1장 |
| 목차 | 1장 |
| 강점 요약 | 1장 |
| 프로젝트 | 각 1~2장 |
| 마무리 | 1장 |

- 표지 / 목차 / 강점 요약 / 마무리는 현재 구성 유지
- 아래 디자인 가이드는 **프로젝트 슬라이드에 적용**

### 목차 슬라이드 구성
- 번호 / 카테고리 / 프로젝트명 / 기간 4개 항목으로 구성
- 카테고리는 작은 뱃지 스타일로 표시 (동그라미 형태)
- 프로젝트명은 카테고리보다 크게 강조
- 카테고리로 역량 영역을 한눈에 파악할 수 있게 설계

## 3. 디자인 기본 세팅

### 슬라이드
- 사이즈: **1920 × 1080** (16:9)
- 여백: 좌우 최소 120px / 상하 80px

### 폰트 (프리텐다드)
- 제목: Pretendard Bold
- 본문: Pretendard Medium
- 강조 숫자: Pretendard Bold

### 폰트 크기

| 요소 | 크기 |
|---|---|
| 제목 | 40–48pt |
| 섹션 제목 (문제/성과 등) | 20–24pt |
| 본문 | 16–18pt (최소 16) |
| 강조 숫자 | 36–48pt |

### 줄간격
- 1.3 ~ 1.5

### 색상 (3개 이하)
- 기본: 블랙 / 화이트
- 강조 성과: 레드 or 포인트 컬러
- 강조 제목: 네이비 계열
- **색 3개 이상 쓰지 않기**

## 4. 공통 레이아웃 구조 (프로젝트 슬라이드)

```
[ 제목 ]

[ 좌측: 문제 / 액션 | 우측: 성과 / 이미지 ]

[ 하단: 인사이트 ]
```

### 정렬
- 기본: 왼쪽 정렬
- 숫자: 중앙 or 강조 박스

### 여백
- 요소 간 간격: 최소 24px 이상
- 블록 간 간격: 40px 이상

### 강조 방식
- ✅ Bold
- ✅ 숫자 크게
- ✅ 박스
- ❌ 색 많이 쓰기
- ❌ 밑줄 남발

### 텍스트 규칙
- 한 줄 = 10~15자
- 문장형 금지 → 키워드형으로

## 5. 슬라이드 콘텐츠 구성 원칙

### 기본 구조 (성과 있는 프로젝트)
```
[ 제목 — 성과 한 줄 요약 ]
[ 핵심 지표 숫자 크게 ]     ← 스캔하는 사람 눈에 바로
[ 문제 | 액션 ]              ← "어떻게?"가 궁금한 사람을 위한 맥락
[ 인사이트 ]
```
- 숫자로 시선을 잡고, 스토리로 납득시키는 구조

### 예외: 흐름형 구조 (실험/검증 중심 프로젝트)
```
[ 제목 ]
[ 문제 → 가설 → 실행 → 결과 ]
[ 인사이트 ]
```
- 성과보다 사고 과정이 강점인 프로젝트에 적용
- **파이썬 3기**에만 적용

## 6. 프로젝트별 슬라이드 구조

### Project 01 · 클라우드 6기 (2장)

**슬라이드 1** — 성과 중심
```
[ 제목 — 성과 한 줄 요약 ]
[ 핵심 지표: CVR 1.93% → 2.86% (+48%) 크게 ]
[ 문제 | 액션 ①②③ ]
[ 인사이트 ]
```

**슬라이드 2** — CRM 실험
```
[ 제목 ]
[ 좌측: 액션 ④ / 인사이트 | 우측: 온사이트 팝업 이미지 크게 ]
```
- 이미지 1장이라 우측에 크게 배치해 여백 활용

---

### Project 02 · 파이썬 3기 (1장) — 흐름형
```
[ 제목 ]
[ 좌측: 문제 → 가설 → 실행 → 결과 / 인사이트 | 우측: 온사이트 팝업 이미지 ]
```
- 이미지 1장, 우측 중앙 배치 + 여백 여유있게

---

### Project 03 · TIL 블로그 챌린지 (1장)
```
[ 제목 — 성과 한 줄 요약 ]
[ 핵심 지표: 참여율 4% → 10% 크게 ]
[ 좌측: 문제 / 액션 ①②③ / 인사이트 | 우측: 팝업 3장 나란히 (작게) ]
```
- 우측 이미지: 상황별 팝업 3종 나란히 배치
- "상황마다 다르게 설계했다"는 걸 시각적으로 보여주는 구조

---

### Project 04 · 핏큘레이터 (1장) — 에이스 슬라이드
```
[ 제목 — 성과 한 줄 요약 ]
[ 핵심 지표: 회원가입률 6배 크게 ]
[ 좌측: 문제 / 액션 / 인사이트 ]
[ 우측 상단: 랜딩페이지 2종 나란히 ]
[ 우측 하단: 이메일 3종 나란히 (Welcome + Resend 2개) ]
```
- 이미지가 많으므로 우측을 이미지 쇼케이스로 활용
- "직접 다 만들었다"가 한눈에 보이는 구조
